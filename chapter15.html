Chapter 15: Layout

This is possibly the most important chapter. Most of the information here is used to teach you how to create a layout for your webpage. Styling blocks into the correct space/slot that you want.

Key Concepts

Before digging in, let's review block and inline elements. Block elements start on a new line and act as the main building blocks. Inline boxes flow between surrounding text. Remember you can use borders, padding, and margins to separate boxes.
When one block element sits inside another block element, the containing element is known as the containing element, or parent element. For example, a p element inside a div element. The div is the parent (or containing) and p is the child.
There are 3 positioning schemes: normal flow, relative positioning, and absolute positioning. You also should know the difference between Fixed Positioning and Floating Elements. Fixed Positioning is a form of absolute positioning in which the position of the element is based on the browser window, as opposed to the containing element. They do not effect the positioning of surrounding elements. Floating Elements allow you to take an element out of normal flow, and position it to the far right or left of a containing box. The floating element becomes a block level element that allows other content to flow around.
Also remember, when moving a box from normal flow, boxes can sometimes overlap. Use the z-index property to control which box appears on top.

Normal Flow

In normal flow, each blcok level sits on top of another. This is the default way that browsers treat HTML. It's unnecessary to specify this in CSS, but if you are ever inclined to do so, the syntax is:
position: static;

Relative Positioning

position: relative; will allow you to move an element where it would have been in relation to normal flow. You can move a box horizontally with left or right properties, and up and down with top and bottom properties. The values can be given in pixels, ems, or percent.

Absolute Positioning

position: absolute; will take the box out of normal flow. This box will no longer have an affect on positioning of other elements. The other boxes will act as if this box is not even there.

Fixed Positioning

position: fixed; is a type of absolute positioning. This will set an element into a fixed position on the browser. An example would be a header that stays at the top of the window when you're scrolling through the page. You still need to use the box offset properties (top, bottom, right, left) to place the box in the proper location.

Overlapping Elements

z-index: is used to give boxes a way to decide which one is on top. A larger number will bring the box closer to the front, while a smaller number will push a box further to the back. Relative, absolute, and fixed positioning can all have issues with boxes overlapping. So if you run into this issue, using z-index will keep your boxes organized.

Flow

The float property allows you to take an element in normal flow and push it as far to the right or the left as possible. Anything else inside the containing element will flow around the floated element. So, the values you can use are left and right. You should also always define a width for the floated element, so that it will not span the width of the page.

Columns

You can use float to make boxes sit side by side as if they're columns. with the following code, you'll be able to create columns that side side by side.

body {
width: 750px;
font-family: Arial, Verdana, sans-serif;
color: #665544;}
p {
width: 230px;
float: left;
margin: 5px;
padding: 5px;
background-color: #efefef;}

Although this code does work, there is one issue. The 4th column will sit underneath the 3rd column. You can fix this by using the clear property. You can give it values of left, right, both, or none. This property allows you to tell a box that it can't touch other boxes on the defined side. For example, since the 4th paragraph sits under the 3rd instead of starting a new column, you can give it a class "clear" and give that class a declaration of clear: left; Now the element will nnot be allowed to touch any other element within the containing element on the left side.
If a containing element only contains floated elements, some browsers will treat it as if it is zero pixels tall. You can fix this by assigning the containing element a few declarations. Add in overflow: auto; as well as width: 100% to fix this. In the book on page 373, you can see a div element that looks like 2 lines because of this issue.

In order to create multiple columns, many pages will achieve this by create multiple div elements. Give each a class name like "column1of2" or whatever you feel is appropriate, then assign each a width property, float property, and margins.

Layout

There are 2 types of layouts, Fixed Width and Liquid.

Fixed Width Layout

Fixed width layouts do not change size based on the size of the screen. Measurements are usually fiven in pixels.
Advantages:

  • Pixel values are accurate at controlling element sizes and positioning.
  • Far greater control with appearance and position of items on the page.
  • You can control length of lines of text regardless of size of user's window.
  • Size of an image will always remain the same relative to rest of page.
Disadvantages
  • Can end up with big gaps at edge of page.
  • If the user's screen is higher res than designer's screen, page can look smaller and text can be more difficult to read.
  • If user increases text size, text might not fit into allotted spaces.
  • Design works best on devices that have similar resolutions to desktops or laptops.
  • Page will often take up more vertical space than a liquid layout.

Liquid Layout

Liquid layouts will change based on the size of the browser window.
Advantages:

  • Pages expand to fill entire browser window, no spaces around the page on larger screens.
  • If user has a small window, the page can contract to fit everything.
  • Design is tolerant of the user setting custom font sizes.
Disadvantages:
  • If you don't control width of sections, design can look different than intended. Unexpected gaps, items squashed together, etc.
  • If user has a wide window, lines of text can become longer and harder to read.
  • If user has a narrow window, words can be squashed together and may only have a few words on each line.
  • If a fixed width item, like an image, is in a box too small to hold it, then the image can overflow onto text.
It's usually best to only let parts of the pages expand or contract. A liquid layout is uaually specified in percentages rather than pixels.Here is an example of a liquid layout. As before, make sure you change browser window size so you can see the effects.

Layout Grids

Many web designers use a grid called the 960 grid. As you can see, this grid makes it much simpler to design a page. Each column is 60 pixels wide, with a 10 pixel margin. This grid helps to create continuity between different pages, helps users predict where to find information on new pages, makes it easier to add new content in a consistent way, and helps people collaborate consistently.
