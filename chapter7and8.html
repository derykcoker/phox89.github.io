Chapter 7: Forms

Here we have a wealth of information that I have not used yet. I imagine a lot of it I will not need to use for a long time, but there's no harm in learning it now (and having this page to reference later.)


Form Structure

<form action="http://www.example.com/subscribe.php" method="get" />
Form controls
</form>
Every form requires an action. The action page is the URL that will receive the information in the form once it's submitted.
The method will be either get(default value if undefined) or post. In general, get will be used for:

  • short forms like search boxes
  • when just retrieving data from a server

Post method will send values in what are known as HTTP Headers. Should use post for:
  • user uploadsing a file
  • longer form
  • sensitive data like passwords
  • adds or deletes information from a database

Text Area

What did you think of this gig?


You can use <textarea>to create an input that will include multiple lines. The code above looks like the following:
<textarea name="comments" cols="20" rows="4">Enter your comments...</textarea>
You don't need to use the rows and columns tags, instead you should use CSS to control the height/width of the text box.

Radio Button

Please select your favorite genre:
Rock Pop Jazz

Input type Radio is used for the above code. Can only select one option, cannot deselect. Looks like the following:
<input type="radio" name="genre" value="rock" checked="checked" /> Rock
You will need to create one line of these for each option, changing only the value each time. The text to appear next to each button will be whatever you type right after the "greater than" bracket.

The <label> tag defines a label for an <input> element. The <label> element does not render as anything special for the user.

Grouping Form Elements

Using fieldset and legendwill allow you to group forms together, and give the fieldset a label. Fieldset should surroung all of the form code, and legend should be placed directly under the opening fieldset tag.

Form Validation

Inside an input tag, you can add required="required" to ensure that the form has been filled in correctly before submitting. This helps reduce server load. Validation should be done with Javascript.

Can use <input type="date" name="depart" /> for a form that only accepts dates.

Search:

Chapter 8: Extra Markup

Most of the information here I already know, but there are a few things to write down that are worth noting.

ID Attribute

ID attribute is known as a global attribute because it can be used on any element. Should always start with an underscore or a letter. An ID should always be unique, especially when styling your page with CSS.

Class Attribute

The Class attribute is similar to the ID attribute, except it can be reused. Think of it as using one class for multiple paragraphs. You would want them all to have blue text, so in CSS you can change one class to have a diferent text.

Block VS Inline Elements

Block elements are elements that are naturally displayed in a block. For instance, if you were to put two p elements in a row, the second would start on a new line. Some block element examples include: p, h1-h6, ul, and li.

Inline elements are elements that will continue on the same lin as neighboring elements. Some examples are: a, b, em, and img.

Div

Although I have been using this tag since I started this site, I should still add a little section on it because of its' importance. The Div element is used to "block" together a group of other elements. Useful for sectioning off things like navigation, a footer, and the main body of your page. With CSS, you can style an entire section when applying a class or id to a div element.

Span

The Span element is usually used in the middle of text to apply a class or id to a section of the text. For instance I could type:
<span class="spanexample">This piece</span> of text only has a span element around the first 2 words, giving them the class "spanexample".
