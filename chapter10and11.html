Chapter 10: Introducing CSS

While I have already gone through the track on Codeacademy, I feel like I would still be better off going through this chapter and rewriting anything thats not painfully obvious.

Style Rules

CSS code looks like the following:
p {
color:white;
}
In this code, p is the selector. color: white; is the declaration. In the declaration, color is a property, while white is a value.

HTML and CSS

There are two ways of adding CSS to an HTML file, you can either link your CSS while with the HTML file, or inject CSS rules directly into the HTML code. If you have multiple pages, it's usually best to create a separate file so that you can reuse your code. This way also makes it easier to make chasnges to code, since you only have to edit one file.
To link the files together, you need to add the fopllowing code inside your head element:
<link href="pathtoCSSfile.css" type="text/css" rel="stylesheet" />
The type specifies the type of document being linked, and the rel specifies the relationship between the two files.
To add CSS code directly into the HTML file, all you have to do is in the header create <style type="text/css"> and </style> elements, with CSS code in the middle.

Selectors

Selectors usually follow a rule that the most specific takes precedent. If you have a rule for all p elements, then a rule for only p elements within a class, the p elements within the class will take priority. Below are most of the selectors and how to use them. You can also specify for one rule to always take precedent, by adding !important to the end, but still before the semicolon.
Universal Selector: Applies to all elements: * {}
Type Selector: Matches element names: h1, h1, h3 {} - targets all h1, h2, and h3 elements.
Class Selector: Targets elements with matching class: .class {} - p.class {} - Targets all classes named "class." Second targets all p elements with class id "class".
ID Selector: Same rules as class, except targets ID: #introduction {}
Child Selector: Element thats direct child of another: li > a {} - targets only <a> elements that are children of <li>.
Descendant Selector: All elements that are descended (child, grandchild, etc): p a {} - Would select all a elements that are nested inside a p element, even if other nested elements between.
Adjacent Sibling Selector: Matches element that is next sibling of another: h1+p {} - Targets only first p element after a h1 element, but not the other p elements.
General Sibling Selector: Matches all siblings, does not have to be directly preceding: h1~p {} - If theres 2 p elements that are siblings of h1, rule would apply to both.

Chapter 11: Colors

This chapter focuses on changing colors. This section will be very brief because most of the information is very self explanatory.


Foreground and Background Colors

Before we get started, here are some very useful links.
Palleton is useful for choosing an entire color scheme.
This tool is useful for checking how well colors contrast with each other.
Using the "color" and "background-color" properties, you can specify color in 4 different ways: Color name, RGB, hex value, or HSL.
Color name: color: DarkCyan;
RGB: background-color: rgb(100, 100, 90);
Hex value: color: #9a87bd;
HSL (hue, saturation light): background-color: hsl(305, 50%, 100%);
HSL numbers are used as follows: 0-360 (color wheel number is the angle), then a percent from 0-100 for staursation, then a percent from 0-100 for lightness.

At the time of this writing, RGB and HSL seem to be the best options because they support alpha channels by use of RGBA and HSLA. To use alpha, you just add a 4th number inside the parenthesis ranging from 0.0 to 1.0. 0.0 being 0% (fully see through) and 1.0 being 100% (completely visible.)
color: rgba(248, 148, 264, 0.4);
background-color: hsla(302, 50%, 25%, 0.75);
