Chapter 12: Text

This chapter covers the different formats and stylings you can use for text.

Typeface

Serif: fonts that have extra details at the ends, easier to read for long passages. Times Roman and Georgia are the most common.
Sans-Serif: fonts that do NOT have extra details at the ends. Can be clearer to read. Helvetica and Arial are the most common.
Monospace: fonts where all the letters have the same width. Useful for coding since the letters will always align together. Courier is a common font in this category.
font-weight: Light, Medium, Bold, Black. Used for emphasis, can also help to show contrast.
font-style: Normal, Italic, Oblique. Italic gives the font a more cursive style, while oblique just angles the font.
font-stretch: Condensed, Regular, Extended. Condenses will make the letters thinner and closer together, while extended makes the letters wider and further apart.

When choosing a font type, a font will only display when the font is already installed on the viewers computer, or unless you add some code (explained later.)

Font Family

Using font-family allows you to specify which font to use on your page. Common practice is to list 3 items separated by commas. The first should be the font you want to use, 2nd a backup font, and 3rd the typeface incase the first 2 are unavailable. This way the viewers browser can choose a font for them.

font-family: Georgia, Times, serif;
font-family: "Courier New", Courier, monospace;

Text Size

There are 3 different methods to choosing a font size: pixels, percent, and em.
font-size: 12px; Pixel is pretty self explanatory. Just choose the px size you want. This is the most common method.
font-size: 75%; Default pixel size is 16 px, so this code would change the font size to 12. However, percentages do chain together. If you were to make all text inside the body element 75% (12 px) and then created an addition rule for p tags to be 75%, then all p elements inside the body element would be 9%.
font-size: 1.5em; Em is useful for text size relative to parent (not used as much). Below are some Examples of how each would look in code.
body = 16px = 100% = 1em
h3 = 18px = 133% = 1.125em
h2 = 24px = 150% = 1.5em
h1 = 32px = 200% = 2em

Font Face

@font-face can be used to add a font to your page that a user does not have installed. This code will tell the user's browser to download the font from the given source. An easy way to do this is to go to This webpage from Google. Once you pick the font you want, you will see a code for you to embed into the head element in your HTML document. Once you do that, you can use the font-family property in your CSS document to choose the font without the need to use @font-family.
However, if you would rather upload your own font, you will need to upload the font in 4 different formats: eot, woff, ttf/otf, svg. Below is an example of how the code would look for a font titled ChunkFiveRegular.
@font-face {
font-family: 'ChunkFiveRegular';
src: url('fonts/chunkfive.eot');
src: url('fonts/chunkfive.eot?#iefix')
format('embedded-opentype'),
url('fonts/chunkfive.woff') format('woff'),
url('fonts/chunkfive.ttf')
format('truetype'),
url('fonts/chunkfive.svg#ChunkFiveRegular')
format('svg');}

Text Transformations and Effects

text-transform: uppercase, lowercase, capitalize;
Capitalizes all letters, all letters are lowercase, and capitalizes the first letter of each word.
text-decoration: none, underline, overline, line-through, blink;
These are all obvious, except blink. Blink causes the word to blink, and is frowned upon due to this being extremely annoying.
line-height: 1.4em;
Line-height is interesting. Best used in em due to the fact that some users will change the text size in browser. This changes the veritcal gap between lines of text. Think of it like spacing from Microsoft Word.
letter-spacing: and word-spacing: .4em;
Kerning is the term used by typographers for the space between each letter. The value given will be added on top of the default value specified by the font. The default value between words is ~0.25em. You probably won't need to change the space between words unless you increase letter spacing, and want to help the text be a little more readable/
text-align: left, right, center, justify;
Aligns text to the left, right, or center. Justify dictates that every line should take up the full width of the box, except the final line. Justified text looks at the words on each individual line and creates an equal gap between those words. Can look odd if you have larger words mixed in. Better to align left most of the time.
vertical-align: baseline, sub, super, top, text-top, middle, bottom, text-bottom.
Not intended to vertically align text in the middle of block elements like p and div, but does have this effect when used in table cells. Its use is more for inline elements like img, em, or strong.
text-indent: 20px; Used to indent the first line of text. Can use a negative value to push text off of the browser window.
text-shadow: 1px 1px 1px #000000;
Takes 4 values: First value = length to the left or right shadow should fall. Second value = distance to top or bottom shadow should fall. Third value (optional) = amount of blue to apply. Fourth value = color. Might not be 100% supported in every browser, when using make sure to test.

Pseudo-Elements and Pseudo-Classes

A pseudo-element acts like an extra element is in the code. Whereas a pseudo-class acts like an extra value for a class attribute. You specify them both at the end of the selector in your CSS code. The first 2 examples below are pseudo-elements, and the rest are pseudo-classes.
first-letter and first-line
used to only apply to the first letter or first line. Code looks like this:
p.intro:first-letter {
font-size: 200%; }
p.intro:first-line {
font-weight: bold; }

:link :visited :hover :active
These are useful for links you have on your page. A common way to use these is for changing the color, adding underlines, etc. Some sample code is below.
a:link {
color: deeppink;
text-decoration: none;}
a:visited {
color: black;}
a:hover {
color: deeppink;
text-decoration: underline;}
a:active {
color: darkcyan;}

:hover :active :focus
Hover is used when the mouse cursor is placed over an item, such as a link or a button. Active is when an element is being activated by the user (like a button.) Focus occurs when a browser discovers that you are ready to interact with an element on the page. For example, when your cursor is in a form input ready to accept typing.
Pseudo classes should be used in the following order:

  • :link
  • :visited
  • :hover
  • :focus
  • :active

Attribute Selectors

Below are a set of attribute selectors that allow you to create rules that apply to elements that have an attribute with a specific value.

Selector Meaning Example
Existence [] - Matches a specific attribute, whatever its value. p.[class] - Targets any p element with an attribute called class.
Equality [=] - Matches a specific attribue with a specific value. p[class="dog"] - Targets any p element with an attribute whose class value is dog.
Space [~=] - Matches a specific attribute whose value appears in a space separated list of words. p[class~="dog"] - Targets any p element with an attribute class whose value is a list of separated words, one of which is dog.
Prefix [^=] - Matches a specific attribute whose value begins with a specific string. p[attr^"d"] - Targets any p element with an attribute whose value begins with "d".
Substring [*=] - Matches a specific attribute whose value contains a specific substring. p[attr*"do"] - Targets any p element with an attribute whose value contains the letters "do".
Suffix [$=] - Matches a specific attribute whose value ends with a specific string. p[attr$"g"] - Targets any p element with an attribute whose value ends with "g".

Chapter 13: Boxes

This chapter covers how boxes work. Padding, margins, etc.

Box Dimension

Using height: and width: you can control the size of your boxes that contain elements. You can specify by pixels, percent, or ems. Percentages size will be used based on the "parent box." So if a p element is set to 75% inside a div element set to 400x400 pixels, the box for the p element would be 300x300 pixels. When using ems, the size of the box depends on the size of the text within. Developers have started to use percentages and ems more lately to accomodate more devices.
You can also use min-width, max-width, min-height and max-height to set minimum/maximum values for boxes. min/max-width are both useful for smaller screens.
If the boxes are set too smal to fit all of the content, you can use the overflow: property. The values you can use are hidden (to hide the overflowing content) or scroll (to add a scrollbar to the box.) You can also use visible, but if the box is already overflowing and overflow is not being used, then there isn't really a point.

Borders, Margins, and Padding

some people can really struggle with this subject. Before explaining, a good example of each can be found here. Basically, border is the outside of the box. Margins refer to the space between neighboring boxes. Padding refers to the space between the border and the content inside the box.
border-width can be specified in a few ways. The first way is to define one pixel size, such as border-width: 5px; You can also thin, medium, or thick. The final way is similar to the first. If you define 2 pixel sizes, then the first will define the top and bottom, and the second the left and right. If you define 4 pixel sizes, then you can define each side. The order is clockwise starting from the top (top, right, bottom, left.)
border-style allows you to give the border a different look. Just simply type border-style: solid; There are a many different styles, which can be seen here.
border-color is self explanatory. It follows the same rules as above as far as coloring each side or altogether, but you must use RGB or a hex value.
You can combine these 3 together into one statement by writing them in the same as order as presented above.
border: 5px dotted #8f7c6a;

Padding and margin are usually specified in pixels, but you can use percentages or ems instead if you would rather. It's important to remember that if you specify a box to be 200 pixels wide, the content will be automatically scaled based on the padding size, and border size. When one box is sitting under another, the larger margin will be used insted of adding them together. So if one box with a 15 pixel margin is sitting above a box with a 25 pixel margin, the space between the boxes will only be 25 pixels.

Styling Content

Centering a box on a page is pretty simple. You just need to set the left and right margins to "auto." When doing this, make sure to set a width for the box, otherwise it will be the width of the entire page. It's a good idea to also add text-align: center; to any text inside a centered box, for older browsers.
display can be used to change inline elements to block, and vice versa. With the display property, you can use the following values: block, inline, none, and inline-block. None hides the block from the page, while inline-block allows block level elements to flow like inline, but still retain block features. Go back to page 317 for more info. You can also use visibility: hidden; to hide a block. When using this method, the space intended for the block wil still show though.
border-image can be used to have a custom image used for the border. You will have to specify 3 things: the url for the image, where to slice the image, and what to do with straight images. You can choose stretch to stretch the image, repeat to repeat it, or round to try to repeat the image, but scaled so it will fit perfectly. The code will look like this following:
border-image: url("imagesource.jpg") 10 10 10 10 round;
You can also add "-moz-" and "-webkit-" as a prefix to border-image to help with compability with earlier browser versions. It's a good idea to have 3 total lines: one with moz, one with webkit, and one normal.
box-shadow functions similar to border-image. It also lends syntax from text-shadow. You must use the first 2 of the 4 total values (horizontal offset, vertical offset, blur distance, shadow spread.) You can also use the inset value at the beginning to create a shadow inside the box. This code will look like box-shadow: 5px 5px 5px 5px #FFF777; As with border-image, it's best to create 3 lines using "-moz-" and "-webkit-"
Rounded and elliptical borders are fairly simple. To round all the corners all you need to do is type border-radius: 5px; (As with the previous 2 lines of code, use moz and webkit.) To create a full circle, you just have to set the border radius to the same height as the box. You can create elipses by specifying each corner individually with border-top-left-radius: or something similar for each corner. When doing this, you can use one value for an evenly rounded corner, or 2 values. When you using 2 values. like 25px and 50px, the corner will round at 50 px wide, but 25 px tall.
You can do this in a shorthand version by specifying 4 horizontal values and 4 vertical values all in one line. This will require further testing on jsfiddle to get a feel for how it works, but the code will look like this: border-radius: 1em 4em 1 em 4em / 2em 1em 2em 1em;
